ToloWiz Help

1. The Menu Bar

File

Provides operations concerning files in your file system like loading and saving ontologies and configurations.

Open Ontology (Ctrl + O)

Choose an Ontology file (rdf or owl) from your file system. The described ontology will now be interpreted and displayed by ToloWiz. The ontology file must be written in RDF/XML syntax.

Import Configuration

Choose a .tolo file from your file system to open and display an ontology, whose view has already been adjusted with ToloWiz.

Export Configuration

Generate a .tolo file in your file system that contains your current ontology as well as all information concerning layout and arrangement of its view. This file can then be shared with others to exchange ontology arrangements.

Export As Image

Generates a png file, showing your whole current ontology like you arranged it with ToloWiz.

Export as PDF

Generates a PDF file, showing your whole current ontology like you arranged it with ToloWiz.

Close Ontology (Ctrl + W)

Closes the currently displayed ontology and its tab in the program.

Graph

Provides operations manipulating the view of your currently opened ontology.

Restore Standard Alignment

Restores the alignment of all nodes and edges of the graph representing the ontology to the default alignment calculated by ToloWiz. This means that all types and relations are hidden initially, you have to check the according boxes again.

Restore Standard View

Restores the default view of the ontology. All edges and nodes of the graph will be shown in the ToloWiz default design again. This includes relation styles, type icons, hidden instances and groups.

Load Configuration

Loads a configuration saved in the internal ToloWiz database. You can pick one from the file chooser. Displays the ontology with the alignment and view described by the configuration.

Save Configuration (Ctrl + S)

Saves the ontology with the current alignment and view in the internal ToloWiz database.

Undo (Ctrl + Z)

Undoes your latest operation changing the ontology graph.

Redo (Ctrl + Y)

Redoes the latest undo.

View

Provides operations to change this program's appearance.

Font Size

Changes the font size of all elements of the user interface. This does not influence the font size in the graph.

Presenting Mode

Changes the view of the program to only show the visualization of the current ontology. It is not possible to show a context menu to a selected node anymore, but you can still zoom, pan or drag a node. Leave the presenting mode by clicking "Leave Presenting Mode".

?

Provides further information about the program.

Help (F1)

Shows this help page to explain basic features of the program.

3. The Ontology Graph

This is the main part of the program. Here, all opened ontologies will be displayed in their respective tabs. There is an interactive ontology graph view as well as undo and redo buttons.

Possible interactive operations:

  • Change the position of instances with drag and drop.
  • Open a detail menu by right-clicking on an instance to show the instance's values or hide it. You can also add it to or remove it from a group. Additionally, you can mark the selected instance with a color, shape or stroke.
  • Change the zoom factor by scrolling with the mouse-wheel. You will zoom in to the point your mouse cursor is currently pointing to.
  • Set the zoom factor so that the whole graph is visible by right clicking to the background and selecting "Show Total Graph".
  • Navigate through the visualization with the arrow keys of the keyboard.

4. Value Area

This area displays values and relations of a selected instance. You can show values of an instance by right-clicking it and selecting "Show Values". You can select which values will be shown for which type in the tab "Values" of the Configuration Area.

5. The Configuration Area

In this part, you can manipulate visibility and style of types, relations and values.

  • Types: After opening an ontology, all types are unchecked and thus invisible. You can show all instances of a certain type by clicking the checkbox next to the type.
    By clicking the arrow to the right of the type, you can open another menu to change the appearance of all instances of the type.

    • Import Icon: Select an icon from your file system that could later be chosen as an icon for this type.
    • Change Icon: Sets a new icon for all instances of this type. The icon needs to be imported first.
    • Change Mark: Change the style of the instances of this type. You can change the shape of the icon, the stroke around the icon and the color of this stroke.
    • Display As...: Choose a way to display the instances of a type. There are currently two ways to display instances: "Default" or "Interface".

  • Relations: after opening the ontology, all relations are unchecked. You can make a certain kind of relation visible by clicking the checkbox next to its name. There is also another menu accessible by clicking the arrow to the right of each relation name. Here, you can change the color, stroke and arrowhead of each relation.

  • Values: here, you can select which values to show for an instance of each type by navigating to the correct type in the type hierarchy and clicking the checkbox next to the value you want to show. The values will be displayed in the Value Area. The selection will only show you the values that exist for instances which have this type as their direct type.

  • Groups: here, you can create a new group of instances with a group name of your choice. You may select certain instances or whole types. This enables you to perform operations on the whole group, like showing or marking, instead of repeating them for every member individually. You can also delete your existing groups.

  • Hidden Instances: Here is a list of instances that you hid by selecting them and choosing the "Hide Instance" menu item. You can show them again by clicking "Show".

6. Error Bar

This area will show warning messages if problems occur during the program execution. You can click on it to get a more detailed description on the errors that occured.
